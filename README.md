# 🏭 Planning de Production — NC-MX & TE-ME

Dashboard interactif de suivi des commandes de production.
**100% navigateur** — aucun serveur, aucune installation.

## 🌐 Accès

👉 **https://TON-PSEUDO.github.io/planning-production/**

## 🚀 Mise en place (5 minutes)

### 1. Créer le dépôt GitHub
- Allez sur github.com → **+** → **New repository**
- Nom : `planning-production`
- Cliquez **Create repository**

### 2. Uploader les fichiers
- Cliquez **"uploading an existing file"**
- Glissez les 3 fichiers : `index.html`, `README.md` et votre fichier Excel renommé **`production.xlsx`**
- Cliquez **Commit changes**

### 3. Activer GitHub Pages
- Allez dans **Settings** → **Pages**
- Source : **main** / **/ (root)**
- Cliquez **Save**
- Attendez 2 minutes

### 4. C'est en ligne ! 🎉
Ouvrez : `https://votre-pseudo.github.io/planning-production/`

## 📊 Mettre à jour l'Excel (chaque semaine)

1. Renommez votre nouveau fichier Excel en **`production.xlsx`**
2. Sur GitHub, cliquez **Add file** → **Upload files**
3. Glissez le nouveau `production.xlsx`
4. Cliquez **Commit changes**
5. Attendez 2-3 minutes (cache GitHub)
6. Vos utilisateurs cliquent **🔄 Actualiser** dans l'app

> GitHub garde l'historique de toutes les versions !

## 📤📥 Partager le suivi entre collègues

Le suivi (commercial, version, plans...) est stocké dans chaque navigateur.
Pour synchroniser entre collègues :

1. **📤 Exporter** : Télécharge un fichier JSON du suivi
2. **Partager** : Envoyez le JSON par email ou dossier partagé
3. **📥 Importer** : Les collègues importent le JSON

## ⚠️ Important

- Le fichier Excel doit s'appeler exactement **`production.xlsx`**
- Il doit contenir les feuilles **TE** et **ME**
- GitHub Pages met ~2-3 min à propager les changements
